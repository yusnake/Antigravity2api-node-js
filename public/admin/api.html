Antigravity API 文档

所有接口均需在请求头中携带 Authorization: Bearer $API_KEY

返回控制台
认证

API Key

所有接口均使用环境变量 API_KEY 校验。

  • 请求头：Authorization: Bearer <API_KEY>
  • Content-Type：对于 POST 请求请使用 application/json
  • 未通过校验将返回 401 Unauthorized
GET /v1/lits

凭证清单

返回所有已启用的凭证名称及当前小时用量与剩余额度。

请求示例

GET /v1/lits
Authorization: Bearer <API_KEY>

响应示例

{
  "credentials": [
    {"name": "project-1", "used_per_hour": 3, "remaining_per_hour": 17},
    {"name": "project-2", "used_per_hour": 0, "remaining_per_hour": 20}
  ],
  "windowMinutes": 60,
  "limitPerCredential": 20,
  "updatedAt": "2025-01-01T00:00:00.000Z"
}
POST /v1/chat/completions

OpenAI 兼容接口

与 OpenAI /v1/chat/completions 兼容，自动轮换凭证。

请求示例

POST /v1/chat/completions
Authorization: Bearer <API_KEY>
Content-Type: application/json

{
  "model": "gemini-pro",
  "stream": true,
  "messages": [
    {"role": "user", "content": "你好"}
  ]
}

响应示例（流式片段）

data: {"id":"chatcmpl-...","object":"chat.completion.chunk",...}
...
data: [DONE]
POST /{credential}/v1/chat/completions

按凭证直连

指定凭证（projectId）执行聊天接口，不受每小时用量限制。

请求示例

POST /project-1/v1/chat/completions
Authorization: Bearer <API_KEY>
Content-Type: application/json

{
  "model": "gemini-pro",
  "messages": [
    {"role": "user", "content": "使用指定凭证回复我"}
  ]
}

响应示例（非流式）

{
  "id": "chatcmpl-...",
  "object": "chat.completion",
  "created": 1735689600,
  "model": "gemini-pro",
  "choices": [
    {
      "index": 0,
      "message": {
        "role": "assistant",
        "content": "好的，这是按指定凭证生成的回复。"
      },
      "finish_reason": "stop"
    }
  ]
}

当凭证不可用或被停用时返回 404，不会计入轮询用量。

  ]
